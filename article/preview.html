测试全绿,但 AI 偷偷把你的系统改坏了

研究Agent的云

上个月读到一个 YC 创业者的复盘,里面有句话我盯着看了很久:

"Claude 自信满满地重构了一个服务边界,所有测试都过了——但它悄悄打破了一个集成契约。"

测试全绿,系统是坏的。

这不是孤例。Karpathy 在他那份被全网疯传的 CLAUDE.md 里点过 AI 写代码的三宗罪:不吭声地做错误假设、过度工程、改了根本不该动的代码。curl 的作者干脆取消了 bug 赏金,因为涌进来的全是低质量 AI 提交。维护 Python 生态一堆库的 Jazzband 社区,今年直接解散了,理由写得明明白白:扛不住 AI 生成的垃圾 PR 和 issue。

还有个更扎心的统计:用 AI 辅助的开发者,提交速度是原来的三四倍,引入安全缺陷的速度是原来的十倍。

为什么测试拦不住

因为 AI 不只改代码,它还改测试。

你以为 CI 是红绿灯,但红绿灯是它装的。我整理了 AI 改代码最常见的六种"测试全绿但实际坏了",每一种都见过真实案例:

一、静默契约漂移。函数签名一个字没动,含义变了。超时参数从秒改成毫秒,文档更新了,调用方没更新——你传的 60 秒,现在是 60 毫秒。

二、越界改动。PR 说是修分页 bug,确实修了,顺手把重试次数从 3 改成 0、把一个公共函数改了名,描述里只字未提。

三、假修复。工单说邮箱校验有 bug,它"修"完之后代码一个字节没变,只是加了条断言永远为真的测试。

四、测试掩蔽。它改了哪个函数,就把哪个函数 mock 掉。测试套件从此测的是 mock,不是代码。

五、吞错误。"修好了偶发崩溃"——打开一看,except Exception: continue。崩溃确实没了,失败的数据也无声无息地丢了。

六、架构违规。CLAUDE.md 里白纸黑字写着"所有持久化必须走 Repository 层",它新写的模块直接 open() 了文件。功能正确,测试全过,规矩碎了。

这六种,后面都会再出现一次——作为我们埋进考场的毒药。

所以我做了 trustdiff

一个 Claude Code skill,专门审 AI 写的 diff。名字就是它要回答的问题:这个 diff,你信得过吗?

它不是又一个代码审查工具。通用审查看的是代码质量;trustdiff 盯的是上面那六种 AI 特有的死法。流程五步:

先把 PR 声称的意图钉死成一句话,后面每一个改动块都要对着这句话过堂。然后跑一个"爆炸半径测绘",列出所有被改的函数和谁在调用它们——diff 只告诉你改了什么,只有调用方能告诉你改坏了什么。接着过一遍 19 点清单,覆盖范围、契约、测试诚实度、错误处理、架构五个维度。每个发现必须给出两段证据:改动在哪一行,被它弄坏的依赖在哪一行,而且必须回答一个问题——"测试套件为什么没拦住它?"答不上来就降级。最后吐一个机器可读的裁决 JSON,能直接卡 CI。

诚实环节:它真的有用吗

做完我没急着写这篇文章,先造了个考场考它。

十个模拟 PR,六个埋毒(就是上面那六种死法,每个都精心做到双边测试全绿),四个干净的当误报陷阱,其中一个故意做成"大面积改动但每一处都合理",专钓乱咬的审查工具。评审 agent 全程盲测,看不到答案。

第一轮,满分。检出 6/6,误报 0/4。

但我留了个对照组:同样的模型,不给工具,就一句"你来审这个 PR"。对照组也满分。

这个结果我们没藏着。它说明在小项目上,前沿模型只要肯下功夫通读,本来就能逮住这些问题。于是把考场放大十倍——一个两千行、十六个模块的电商后端,毒药埋进跨包调用链,外面裹上几十行正经改动当干草。再测,两边还是满分。

说人话:清单并没有让聪明模型更聪明,它只是把"最认真的审法"固化了下来。两边连找到的 blocker 行号都一模一样,这反而验证了清单本身是对的。

真正的增量出现在第三轮。

换便宜模型。现实里跑自动化代码审查的,从来不是最贵的旗舰,是 Haiku 这个档位的快枪手。同一个两千行考场,两臂再测一次:

裸 Haiku:检出 83%,误报 25%。它漏掉了那个夹带的"重试次数 3 改 0",还把一个完全干净的重构误杀了。带上 trustdiff 的同一个 Haiku:检出 100%,误报 0。漏掉的它抓回来了,误杀的它放行了——清单强制它去扫配置改动,反误报规则逼它把等价性论证做完。

一句话总结:trustdiff 不是让聪明模型更聪明,是让便宜模型干出旗舰的活。而且是双向的——既不漏检,也不乱咬。

整个考场、埋的毒药、标准答案、计分脚本,全部在仓库里。你可以自己跑一遍验证我说的每个数字。同类工具里,敢把考卷和答案一起开源的,我还没见过第二家。

```bash git clone https://github.com/duliangkuan/trustdiff cp -r trustdiff/skill/trustdiff ~/.claude/skills/trustdiff ```

然后在任何仓库里对 Claude Code 说一句:"用 trustdiff 审一下这个 PR"。
